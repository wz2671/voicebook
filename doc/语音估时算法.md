# 语音估时算法

基于 TTS 真实录制数据校准，推导精读讲稿的朗读时长估算公式。

## 基准数据

两条真实录制数据来自同一讲书人、同一 TTS 引擎：

| 文件 | 总字符数 | 实际时长 | 实际秒数 |
|------|---------|---------|---------|
| 原理1-5 | 5,457 | 17分14秒 | 1034 |
| 原理6-10 | 4,992 | 15分36秒 | 936 |
| 合计 | 10,449 | 32分50秒 | 1970 |

## 字符构成

将每个文件的字符拆分为三类：朗读字符（中文+英文数字）、标点符号、段落数。

| 成分 | 原理1-5 | 原理6-10 | 占比 |
|------|--------|--------|------|
| 朗读字符 | 4,818 | 4,343 | ~87.7% |
| 标点符号 | 491 | 481 | ~9.3% |
| 段落数 | 52 | 49 | ~9.6 段/千字 |
| 其他（空格等） | 43 | 65 | ~1.0% |
| **总字符** | **5,457** | **4,992** | 100% |

朗读字符占比在两篇文章中分别为 88.3% 和 87.0%，偏差仅 1.3 个百分点。标点占比分别为 9.0% 和 9.6%，偏差 0.6 个百分点。千字段落数分别为 9.5 和 9.8，偏差 0.3。这说明精读讲稿这一体裁的内部比例高度稳定。

## 计算过程

### 第一步：估算三项系数

假设朗读时长由三部分组成：

```
总时长（秒）= 朗读字符 × a + 标点符号 × b + 段落数 × c
```

其中 a 是每字朗读时间，b 是每个标点的额外停顿，c 是每个段落间隙的停顿。

代入两条数据，得到方程组：

```
4818a + 491b + 52c = 1034
4343a + 481b + 49c =  936
```

三个未知数、两个方程，需要先固定一个变量。段落间隙 c 在 TTS 中通常在 0.3~0.8 秒之间，取 c = 0.5。

两式相减消去常数项：

```
475a + 10b + 3c = 98
```

代入 c = 0.5：

```
475a + 10b = 96.5
```

尝试不同的 a 值求解 b，并回代验证：

| a (秒/字) | b (秒/标点) | 原理1-5 估算 | 原理6-10 估算 | 总误差 |
|-----------|------------|-------------|-------------|--------|
| 0.190 | 0.179 | 1030 | 936 | -4s / 0s |
| 0.192 | 0.161 | 1030 | 936 | -4s / 0s |
| 0.195 | 0.134 | 1031 | 936 | -3s / 0s |
| 0.200 | 0.089 | 1033 | 936 | -1s / 0s |

a 在 0.19~0.20 之间、b 在 0.09~0.18 之间都能得到良好的拟合效果。取整后确定：

```
a = 0.19 秒/字   （约每分钟 316 字）
b = 0.18 秒/标点 （每个标点额外停顿）
c = 0.50 秒/段   （每个段落间隙）
```

完整的三项模型为：

```
总时长（秒）= 朗读字符 × 0.19 + 标点符号 × 0.18 + 段落数 × 0.5
```

### 第二步：验证三项模型

| 文件 | 实际时长 | 三项模型估算 | 误差 |
|------|---------|------------|------|
| 原理1-5 | 1034秒 | 1030秒 | -4秒 |
| 原理6-10 | 936秒 | 936秒 | 0秒 |

误差在 4 秒以内，约 0.4%。

### 第三步：推导简化公式

三项模型虽然精确，但需要分别统计朗读字符、标点符号和段落数，实际使用不便。

关键发现：两条数据中，朗读字符占比（87.7%）、标点占比（9.3%）、千字段落数（9.6）几乎相同。这意味着在精读讲稿体裁内，**总字符数已经隐含了标点和段落的停顿信息**——不需要拆开算。

用总字符数直接拟合：

```
平均速率 = 总字符数 / 总时长 = 10449 / 1970 = 5.30 字符/秒
```

即：

```
估算时长（分钟）= 总字符数 / 318
估算时长（秒）  = 总字符数 / 5.30
```

### 第四步：验证简化公式

| 文件 | 总字符 | 实际时长 | 简化公式 | 误差 |
|------|--------|---------|---------|------|
| 原理1-5 | 5,457 | 17分14秒 | 17.1分 | -0.1分 |
| 原理6-10 | 4,992 | 15分36秒 | 15.7分 | +0.1分 |

误差均在 ±0.1 分钟（±6 秒）以内。

## 结论

**精读讲稿的朗读时长 = 总字符数 / 318 分钟。**

318 字/分钟这个速率综合了：
- 纯朗读速率约 316 字/分钟
- 标点停顿约占总时间的 8%
- 段落停顿约占总时间的 2.5%

三项叠加后，整体速率为 318 字/分钟。

简化公式之所以准确，不是因为忽略了标点和段落，而是因为精读讲稿体裁内这三者的比例天然稳定，总字符数本身就是三者的加权代理变量。若将来文章体裁发生显著变化（如纯对话、诗歌、大量列表），则需回到三项模型重新校准。
