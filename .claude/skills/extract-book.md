# extract-book — 书籍章节自动提取

当用户提供一个书籍文件路径时，自动检测格式并调用对应的解析工具，将章节提取到 `chapter/<书名>/` 目录下。

## 支持的格式

| 格式 | 解析器 | 状态 |
|------|--------|------|
| `.mobi` | `mobi_handler.mobi_extractor` | 已支持 |
| `.epub` | `epub_handler.epub_extractor` | 已支持 |

## 执行流程

1. 确认用户提供的文件路径存在
2. 检查文件后缀名是否在支持的格式列表中
3. 如果格式不支持：明确告知用户当前缺少该格式的解析工具
4. 如果格式支持：执行 `python -m src.main extract "<文件路径>"` 提取章节
5. 向用户报告提取结果（章节数量、输出目录、前几个章节标题）

## 错误处理

- 文件不存在 → 提示用户检查路径
- 格式不支持 → 明确列出当前支持的格式，告知用户功能欠缺
- 提取失败 → 输出错误信息，不静默失败

## 示例

用户输入：
```
/extract-book books/游戏设计的100个原理.epub
```

或直接说：
```
帮我提取 books/三体.mobi 的章节
```

执行：
```bash
cd <项目根目录> && python -m src.main extract "<用户提供的路径>"
```

## 注意事项

- 章节输出到 `chapter/<书名>/` 目录，格式为 `chapter1.md`, `chapter2.md` ...
- 每个 `.md` 文件第一行为 `# 章节标题`
- 如果 chapter 目录下已存在同名书籍目录，提取会覆盖已有文件
